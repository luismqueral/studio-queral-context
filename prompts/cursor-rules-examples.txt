Additional Cursor Rules Examples for NYT Design System

## Component-Specific Rules

### Article Components
- Article headlines must use Cheltenham Bold (fonts/cheltenham/cheltenham-normal-700.woff2)
- Article bylines should use Franklin Medium (fonts/franklin/franklin-normal-500.woff2) 
- Article body text uses Imperial Regular (fonts/imperial/imperial-normal-400.woff2)
- Apply proper line-height from typography tokens for optimal readability
- Include article metadata with consistent date formatting

### Navigation Components
- Navigation items use Franklin Semibold (fonts/franklin/franklin-normal-600.woff2)
- Implement sticky header behavior with proper z-index layering
- Include NYT logo with brand-compliant sizing and positioning
- Use semantic nav elements with proper ARIA landmarks
- Implement responsive menu collapse for mobile devices

### Form Components  
- Form labels use Franklin Medium with consistent spacing
- Input fields have 44px minimum touch target for mobile accessibility
- Error states use semantic error colors from design tokens
- Success states use semantic success colors with proper contrast
- Include proper form validation with accessible error messaging

## Responsive Design Rules
- Mobile breakpoint: 320px minimum width
- Tablet breakpoint: 768px using NYT Tachyons breakpoint classes
- Desktop breakpoint: 1024px with max-width container
- Use fluid typography scaling between breakpoints
- Implement touch-friendly interaction areas (minimum 44px)

## Accessibility Rules
- All interactive elements must have focus indicators using brand colors
- Color contrast ratios must meet WCAG AA standards (4.5:1 for normal text)
- Include skip navigation links for keyboard users
- Use semantic HTML headings in proper hierarchical order
- Provide alternative text for all images and graphics
- Implement proper ARIA labels for complex UI components

## Performance Rules
- Preload critical fonts using <link rel="preload"> for above-the-fold content
- Use font-display: swap for non-critical fonts
- Implement lazy loading for images below the fold
- Minimize CSS specificity to improve render performance
- Use CSS containment for isolated component rendering
- Bundle and compress assets for optimal delivery
